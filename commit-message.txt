test(migration): complete academia migration from profile-page to agora-next microservice

This commit finalizes the migration of the academia section from the monolithic
profile-page application to the dedicated agora-next microservice frontend.

MIGRATION SUMMARY:
==================

All pages from profile-page/src/app/[locale]/(main)/academia have been successfully
migrated to agora-next/src/app/[locale] with the following structure:

Migrated Pages:
- / (Home/Academia index) - Landing page with latest exercises and subscription CTA
- /tarifas - Subscription plans page with pricing comparison
- /temas-ejercicios - Tags/themes listing page
- /temas-ejercicios/[tema] - Filtered exercises by specific tag
- /ejercicios - Main exercises listing page with pagination
- /ejercicios/[...slug] - Individual exercise detail pages with MDX support

The routing has been simplified as agora-next is dedicated to the academia/marketplace
functionality, eliminating the need for the /academia prefix in URLs.

ARCHITECTURE IMPACT:
====================

This migration completes the microservices architecture transition:
- profile-page (monolith) → Deprecated
- profile-nest (backend) → API services on port 3001
- profile-next (public profile frontend) → Port 3004
- agora-next (academia/marketplace frontend) → Port 3002

The agora-next service now operates independently while consuming the
profile-nest backend for authentication and user management.
